# Как добавить бота на сервер Discord

## Проблема: Команды не появляются

Если команды бота не появляются в списке slash-команд Discord, это означает, что бот был добавлен без правильных разрешений.

## Решение

### Шаг 1: Получите Client ID вашего бота

1. Откройте [Discord Developer Portal](https://discord.com/developers/applications)
2. Выберите ваше приложение (бота)
3. Перейдите в раздел "OAuth2" → "General"
4. Скопируйте **Client ID** (это длинное число)

### Шаг 2: Создайте правильную ссылку для приглашения

Замените `YOUR_CLIENT_ID` на ваш Client ID в этой ссылке:

```
https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=3165184&scope=bot%20applications.commands
```

**ВАЖНО:** Обратите внимание на `scope=bot%20applications.commands` - это ключевая часть!

### Пример

Если ваш Client ID: `1489648554778820700`, то ссылка будет:

```
https://discord.com/api/oauth2/authorize?client_id=1489648554778820700&permissions=3165184&scope=bot%20applications.commands
```

### Шаг 3: Переустановите бота

1. **Удалите бота с сервера** (если он уже добавлен):
   - Настройки сервера → Интеграции → Боты и приложения
   - Найдите вашего бота и удалите его

2. **Добавьте бота заново** используя новую ссылку:
   - Откройте ссылку из Шага 2 в браузере
   - Выберите сервер
   - Нажмите "Авторизовать"

### Шаг 4: Перезапустите бота

После переустановки бота на сервер, перезапустите Docker контейнер:

```bash
docker-compose restart
```

### Шаг 5: Проверьте команды

1. Откройте Discord
2. Перейдите на сервер, где установлен бот
3. В любом текстовом канале начните вводить `/`
4. Вы должны увидеть список команд вашего бота с иконкой

## Список команд

После правильной установки вы увидите эти команды:

- `/play` - Включить музыку по названию или ссылке
- `/pause` - Поставить на паузу
- `/resume` - Продолжить воспроизведение
- `/skip` - Пропустить трек
- `/stop` - Остановить воспроизведение
- `/queue` - Показать очередь треков
- `/nowplaying` - Что сейчас играет
- `/volume` - Установить громкость (0-100)
- `/clear` - Очистить очередь
- `/commands` - Показать все команды

## Разрешения бота

Число `3165184` в ссылке включает следующие разрешения:

- **View Channels** - Просмотр каналов
- **Send Messages** - Отправка сообщений
- **Embed Links** - Встраивание ссылок
- **Read Message History** - Чтение истории сообщений
- **Connect** - Подключение к голосовым каналам
- **Speak** - Воспроизведение аудио

## Устранение проблем

### Команды всё ещё не появляются

1. Убедитесь, что в ссылке есть `scope=bot%20applications.commands`
2. Полностью удалите бота с сервера и добавьте заново
3. Перезапустите Docker контейнер
4. Подождите 1-2 минуты (Discord может кэшировать команды)

### Бот не отвечает на команды

1. Проверьте, что бот онлайн (зелёный статус)
2. Проверьте логи Docker: `docker-compose logs -f`
3. Убедитесь, что в настройках бота включен **Message Content Intent**:
   - Discord Developer Portal → Bot → Privileged Gateway Intents
   - Включите "Message Content Intent"

### Бот не подключается к голосовому каналу

1. Убедитесь, что вы находитесь в голосовом канале
2. Проверьте, что у бота есть разрешения "Connect" и "Speak"
3. Проверьте логи на наличие ошибок
