# Discord Music Bot 🎵

Discord бот для воспроизведения музыки в голосовых каналах с поддержкой YouTube, SoundCloud и прямых аудио ссылок.

## Возможности

- 🎵 Воспроизведение музыки из YouTube, SoundCloud и прямых ссылок
- 🔍 Поиск треков по названию
- 📋 Управление очередью воспроизведения
- ⏯️ Полное управление воспроизведением (пауза, возобновление, пропуск, остановка)
- 🔊 Регулировка громкости (0-100)
- 📊 Отображение метаинформации о треках
- 📝 Поддержка плейлистов (до 50 треков)
- ⏱️ Автоматическое отключение при бездействии

## Требования

- Python 3.10 или выше
- FFmpeg (установлен и доступен в PATH)
- Discord Bot Token с необходимыми разрешениями
- Стабильное интернет-соединение

## Установка FFmpeg

### Windows

1. Скачайте FFmpeg с официального сайта: https://ffmpeg.org/download.html
2. Распакуйте архив в удобную директорию (например, `C:\ffmpeg`)
3. Добавьте путь к папке `bin` в переменную окружения PATH:
   - Откройте "Система" → "Дополнительные параметры системы"
   - Нажмите "Переменные среды"
   - В разделе "Системные переменные" найдите `Path` и нажмите "Изменить"
   - Добавьте путь к папке `bin` (например, `C:\ffmpeg\bin`)
4. Перезапустите командную строку и проверьте установку: `ffmpeg -version`

### macOS

Используйте Homebrew:
```bash
brew install ffmpeg
```

### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install ffmpeg
```

### Linux (Fedora)

```bash
sudo dnf install ffmpeg
```

## Установка бота

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости Python:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Отредактируйте `.env` и добавьте ваш Discord Bot Token:
```env
DISCORD_TOKEN=your_actual_bot_token_here
```

## Создание Discord бота

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Нажмите "New Application" и дайте боту имя
3. Перейдите в раздел "Bot" и нажмите "Add Bot"
4. Скопируйте токен бота (кнопка "Copy" под именем бота)
5. Вставьте токен в файл `.env`

## Необходимые разрешения Discord

При добавлении бота на сервер, убедитесь, что у него есть следующие разрешения:

### Text Permissions
- **Read Messages/View Channels** - Чтение сообщений и просмотр каналов
- **Send Messages** - Отправка сообщений
- **Embed Links** - Встраивание ссылок (для отображения метаданных)
- **Read Message History** - Чтение истории сообщений

### Voice Permissions
- **Connect** - Подключение к голосовым каналам
- **Speak** - Воспроизведение аудио в голосовых каналах

### Privileged Gateway Intents

В настройках бота на Discord Developer Portal включите:
- **Message Content Intent** - Для чтения содержимого команд

### Ссылка для приглашения бота

Используйте следующий URL для приглашения бота (замените `YOUR_CLIENT_ID` на ID вашего приложения):

```
https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=3165184&scope=bot
```

Где `3165184` - это комбинация необходимых разрешений:
- View Channels
- Send Messages
- Embed Links
- Read Message History
- Connect
- Speak

## Использование

### Основные команды

- `/play <запрос или URL>` - Воспроизвести трек по названию или ссылке
- `/pause` - Приостановить воспроизведение
- `/resume` - Возобновить воспроизведение
- `/skip` - Пропустить текущий трек
- `/stop` - Остановить воспроизведение и очистить очередь
- `/queue` - Показать очередь треков
- `/nowplaying` - Показать информацию о текущем треке
- `/volume <0-100>` - Установить громкость
- `/clear` - Очистить очередь (кроме текущего трека)
- `/commands` - Показать список всех команд

### Примеры использования

```
/play Never Gonna Give You Up
/play https://www.youtube.com/watch?v=dQw4w9WgXcQ
/play https://soundcloud.com/artist/track
/volume 75
/skip
```

## Конфигурация

Настройки бота можно изменить в файле `.env`:

- `DISCORD_TOKEN` - Токен Discord бота (обязательно)
- `COMMAND_PREFIX` - Префикс команд (по умолчанию: `/`)
- `MAX_QUEUE_SIZE` - Максимальный размер очереди (по умолчанию: 100)
- `DISCONNECT_TIMEOUT` - Таймаут отключения при бездействии в секундах (по умолчанию: 300)
- `PAUSE_TIMEOUT` - Таймаут отключения при паузе в секундах (по умолчанию: 600)
- `MAX_PLAYLIST_SIZE` - Максимальное количество треков из плейлиста (по умолчанию: 50)
- `SEARCH_TIMEOUT` - Таймаут поиска в секундах (по умолчанию: 10)
- `EXTRACT_TIMEOUT` - Таймаут извлечения URL в секундах (по умолчанию: 3)

## Устранение неполадок

### Бот не подключается к голосовому каналу

- Убедитесь, что вы находитесь в голосовом канале перед отправкой команды `/play`
- Проверьте, что у бота есть разрешения "Connect" и "Speak"

### Ошибка "FFmpeg not found"

- Убедитесь, что FFmpeg установлен и доступен в PATH
- Проверьте установку командой: `ffmpeg -version`
- Перезапустите терминал после установки FFmpeg

### Бот не отвечает на команды

- Проверьте, что в настройках бота включен "Message Content Intent"
- Убедитесь, что у бота есть разрешение "Read Messages/View Channels"

### Ошибки воспроизведения

- Проверьте интернет-соединение
- Убедитесь, что видео/трек доступны в вашем регионе
- Некоторые видео могут быть заблокированы для извлечения

## Архитектура

Проект состоит из следующих компонентов:

- `main.py` - Точка входа и инициализация бота
- `music_commands.py` - Обработчик команд (Cog)
- `music_player.py` - Управление воспроизведением
- `track_queue.py` - Управление очередью треков
- `track.py` - Класс трека с метаданными
- `audio_source_handler.py` - Извлечение аудио из источников
- `error_handler.py` - Обработка ошибок
